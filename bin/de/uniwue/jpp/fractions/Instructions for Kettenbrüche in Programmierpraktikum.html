Kettenbrüche

In dieser Aufgabe werden wir ein kleines Framework implementieren, um (Ketten-)brüche auf die Kommandozeile auszugeben.

Allgemeine Hinweise

  • Verwenden Sie unter keinen Umständen System.exit()! Dies beendet nicht nur Ihr Programm, sondern auch die Testumgebung des PABS-Servers. Lösungen, die System.exit() verwenden, können nicht abgegeben werden und werden mit 0 Punkten bewertet.
  • Verwenden Sie nie statische Felder, die nicht final sind!
  • Erstellen Sie alle Ihre Code-Dateien mit UTF-8 Codierung.
  • Zeilenumbrüche sind generell mit "\n" oder println() zu erzeugen.
  • Beachten Sie, dass zur Zeit aus technischen Gründen leider nur das Java Feature-Level 1.8 unterstützt wird. Funktionen aus Java 9 können nicht verwendet werden.
  • Lesen Sie in der Java API Dokumentation nach! Nicht nur, wenn das in der Aufgabenstellung explizit gefordert wird, sondern immer, wenn Sie eine Klasse verwenden, die Sie nicht auswendig kennen.
  • Eine von PABS als falsch erkannte Abgabe ist falsch. Die Umkehrung gilt nicht!
  • Jede testbare Abgabe wird vom PABS System als "accepted" gekennzeichnet werden. Dies bedeutet nicht, dass Sie volle Punktzahl (oder überhaupt Punkte) auf diese Abgabe bekommen. Ausschlaggebend sind ausschließlich Art und Anzahl bestandener Tests, sowie eventuell weiterer unsichtbarer Tests. 

Ziel der Augabe

Ziel der Aufgabe ist es, ein vollständiges Framework zu entwickeln, mit dem beliebig komplexe Kettenbrüche und ähnliche Terme sauber auf die Kommandozeile ausgegeben werden können.

Beispiel: Dem Programm wird die folgende Eingabe übergeben:

Implementierung

Zur Implementierung des Frameworks machen wir uns das Entwurfsmuster Kompositum zu Nutze. Als gemeinsames Interface für alle Elemente definieren wir das Interface Element.

Alle Klassen sollen public im Paket de.uniwue.jpp.fractions definiert werden.

Hinweis: In dieser Aufgabe werden verschiedene Sonderzeichen verwendet. Um Darstellungsfehler zu vermeiden achten Sie bitte darauf, dass in Ihrer IDE eine Schriftart mit Unicode Support eingerichtet ist.

Das Interface Element

Erstellen Sie das Interface Element. Erstellen Sie ausschließlich die folgenden Methoden.

  • public int width();: Gibt die Breite des Elements in Zeichen aus.
  • public int lineCount();: Gibt aus, wie viele Zeilen das Element umfasst.

Wenn Sie dem Interface weitere Methoden hinzufügen wollen, müssen diese als static oder default definiert werden.

Die Klasse Term

Erstellen Sie die Klasse Term, die das Interface Element implementiert. Term stellt einen beliebigen einzeiligen Term dar, der innerhalb unserer Brüche verwendet werden kann.

Erstellen Sie die folgenden Konstruktoren.

  • public Term(): Erstellt einen leeren Term.
  • public Term(String term): Erstellt einen neuen Term, der die erste Zeile des übergebenen Strings umfasst. Eventuelle weiter Zeilen werden verworfen. Ist der Parameter null, soll eine NullPointerException geworfen werden.

Überschreiben Sie die Methode toString, sodass sie den im Konstruktor definierten String ausgibt.

Überschreiben Sie die Methoden equals und hashCode, sodass zwei Term Objekte als gleich angesehen werden, wenn sie den gleichen Term beschreiben.

Die Klasse Fraction

Erstellen Sie die Klasse Fraction, die das Interface Element implementiert. Fraction stellt einen einzelnen Bruch in unserem Kettenbruchsystem dar. Als Zähler und Nenner können beliebige andere Elemente eingesetzt werden.

Erstellen Sie den folgenden Konstruktor.

  • public Fraction(Element numerator, Element denominator): Erstellt einen neuen Bruch mit dem gegebenen Zähler und Nenner. Ist einer der Parameter null, soll eine NullPointerException geworfen werden. Hat einer der Parameter die Breite 0, soll eine IllegalArgumentException mit aussagekräftiger Fehlermeldung geworfen werden.

Überschreiben Sie die Methode toString. Der Bruch wird dabei wie folgt zusammengesetzt.

  • Bestimmen Sie die Gesamtbreite des Bruchs. Der Bruchstrich soll das breitere Element vorne und hinten jeweils um ein Zeichen überragen.
  • Das gesamte Element soll eine rechteckige Fläche einnehmen. Sowohl der Bereich vor (links von) als auch hinter (rechts von) den Elementen soll wenn nötig aufgefüllt werden.
  • Zähler und Nenner sollen mittig platziert werden. Geht dies nicht auf, soll der vordere (linke) Füllbereich kleiner ausfallen als der hintere (rechte).
  • Als Füllzeichen ist das Leerzeichen (SPACE U+0020) zu verwenden. Der Bruchstrich soll aus zeichenbreiten Querstrichen (EM DASH U+2014) bestehen.

Beispiel: Der Bruch 1/2x sähe folgendermaßen aus.
   

Überschreiben Sie die Methoden equals und hashCode, sodass zwei Fraction Objekte als gleich angesehen werden, wenn sie den gleichen Zähler und Nenner haben.

Die Klasse Operator

Erstellen Sie die Klasse Operator, die das Interface Element implementiert. Operator stellt einen Operator, wie z.B. Plus oder Mal, zwischen zwei Elementen dar. Der Operator ist eine beliebige, einzeilige Zeichenkette. Die Operanden sind wiederum Elemente.

Erstellen Sie den folgenden Konstruktor.

  • public Operator(Element left, String operator, Element right): Erstellt einen neues Operatorelement mit dem gegebenen Operator und den gegebenen Operanden. Ist einer der Parameter null, soll eine NullPointerException geworfen werden. Hat operator mehrere Zeilen sollen alle bis auf die Erste verworfen werden. Ein leerer Operator oder leere Operanden (Operanden der Breite 0) sind erlaubt.

Überschreiben Sie die Methode toString. Der Operator wird dabei wie folgt zusammengesetzt.

  • Das gesamte Element soll eine rechteckige Fläche einnehmen. Sowohl der Bereich über als auch unter den Elementen soll wenn nötig aufgefüllt werden. Verwenden Sie als Füllzeichen das Leerzeichen (SPACE U+0020).
  • Die Operanden und der Operator sollen vertikal mittig platziert werden. Geht dies nicht auf, soll der untere Füllbereich kleiner ausfallen als der obere.
  • Beachten Sie, dass sowohl der Operator als auch die Operanden leer sein (die Breite 0 haben) können.

Beispiel: Ein Operator aus den Brüchen 1/x und 1/y sowie dem Operator " + " sähe folgendermaßen aus.
   

Überschreiben Sie die Methoden equals und hashCode, sodass zwei Operator Objekte als gleich angesehen werden, wenn Operator und Operanden übereinstimmen.

Die Klasse Parser

Erstellen Sie die Klasse Parser. Sie dient dazu, Kettenbrüche einzulesen und in unsere interne Darstellung umzuwandeln.

Erstellen Sie die folgende Methode.

  • public static Element parse(String input): Wandelt den übergebenen, codierten Kettenbruch in ein Element Objekt um.

Um Kettenbrüche aus Nutzereingaben oder einer Datei einzulesen, braucht es ein einfaches Format, in dem die Brüche codiert sind. Wir orientieren uns dabei an der gewohnten Schreibweise vom Brüchen, wie man sie z.B. von Taschenrechnern kennt.

Ein Bruch wird durch das Schrägstrich-Zeichen (SOLIDUS U+002F) gekennzeichnet. Zusätzlich werden runde Klammern (LEFT PARENTHESIS U+0028 und RIGHT PARENTHESIS U+0029) zur Gruppierung von Ausdrücken verwendet.

Da alle anderen Rechenzeichen für unser Programm bloßer Text sind, haben Bruchzeichen immer die niedrigste Präzedenz. Regeln wie Punkt vor Strich gelten also nicht. Der Bruch "1/x-5" ist entsprechend gleichbedeutend zu "1/(x-5)". Befinden sich mehrere Bruchzeichen auf der selben Ebene wird wie gewohnt von links nach rechts ausgewertet. Der Bruch "1/x/y" ist also gleichbedeutend zu "(1/x)/y".

Whitespace Zeichen (insbesondere Leerzeichen) am Anfang und Ende eines Ausdrucks sowie Klammern, die einen vollständigen Ausdruck umschließen, werden nicht dargestellt. Achtung! Dies bedeutet nicht, dass alle Klammern und Leerzeichen entfernt werden. Im Bruch "x + sin(x) / (1/x)" bleiben beispielsweise die Leerzeichen um das Plus sowie die Klammern des Sinus erhalten. Lediglich die Klammern um den Ausdruck 1/x werden entfernt sowie die Leerzeichen vor und nach dem Bruchzeichen.

Zur Auswertung der Eingaben sollen Sie folgenden Algorithmus umsetzen.

  1. Whitespace Zeichen vor und hinter dem Ausdruck werden entfernt.
  2. Enthält der Ausdruck kein Bruchzeichen, ist er ein Term. Dies gilt auch für den leeren Ausdruck.
  3. Ist der Ausdruck von zusammengehörigen Klammern umschlossen, werden die Klammern entfernt. (Gegenbeispiel: Im Ausdruck "(1/x)*(1/y)" werden keine Klammern entfernt. Erstes und letztes Zeichen sind zwar Klammern, gehören aber nicht zusammen.)
  4. Hat ein Bruchzeichen die niedrigste Präzedenz, ist der Ausdruck ein Bruch. Der Ausdruck vor dem Bruchzeichen ist der Zähler, der nach dem Bruchzeichen der Nenner.
  5. Andernfalls ist der Ausdruck ein Operator. Beim Aufteilen des Ausdrucks können Sie sich zu Nutze machen, dass mindestens einer der Operanden von Klammern umgeben ist. Befinden sich nur auf einer Seite Klammern, wie z.B. bei "sin(1/x)", ist der komplette ungeklammerte Teil als Operator anzusehen (im Beispiel also "sin").

In folgenden Fällen ist die Eingabe ungültig und die Auswertung wird mit einer IllegalArgumentException abgebrochen.

  • Die Anzahl der öffnenden und schließenden Klammern stimmt nicht überein. (Beispiel: "(1/x/(1/y)")
  • Der Zähler oder Nenner eines Bruches ist leer. (Beispiel: "1 / / x")

Alle anderen Eingaben sollen von Ihrem Programm verarbeitet werden können.
Testen Sie dies ausgiebig!

Beispiel

Der folgende Ausdruck soll von unserem Programm geparsed werden: "sin(x) + (1/x) / x + 7".

Als erstes ist der zweite Bruchstrich (16. Zeichen) zu betrachten. Da er nicht von Klammern umschlossen ist, hat er automatisch die niedrigste Präzedenz und wird entsprechend als erstes geparsed (Regel 4). Der Ausdruck ist entsprechend ein Bruch. Als nächstes parsen wir den Nenner des Bruches, also " x + 7". Nach Regeln 1 und 2 wird daraus der Term "x + 7". Bleibt noch der Zähler des äußeren Bruchs "sin(x) + (1/x) ". Auch hier greift wieder Regel 1 und dann Regel 5, da der Bruchstrich in Klammern steht. Als Operator identifizieren wir den Ausdruck zwischen den beiden geklammerten Blöcken " + " (incl. Leerzeichen). Als linke Seite gilt es "sin(x)" zu parsen, was nach Regel 2 ein Term wird. Als rechte Seite bleibt "(1/x)", das nach den Regeln 3, 4 und 2 in einen Bruch aus den Termen "1" und "x" zerlegt wird.

Das Ergebnis sieht dan folgendermaßen aus.
